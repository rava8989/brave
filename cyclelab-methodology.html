CycleLab — Complete Methodology

SPX intraday cyclicality, with its own negative results attached · ← back to CycleLab

1 · What this tool is — and the finding that defines it

CycleLab asks whether the intraday shape of SPX sessions repeats: does Monday behave like prior Mondays, hour by hour? It composites the last N same-weekday sessions into a prediction line and scores that line against reality, every day, permanently.

The headline result is negative, and the page says so on its face: the same-weekday composite has zero out-of-sample directional signal. Tested four independent ways: (a) direction hit-rate ≈ always-up baseline (53.6% vs 53.6%); (b) no averaging method × lookback combination beat a flat-zero baseline on end-of-day delta by more than 1%; (c) a wrong-weekday control — feeding the model deliberately mismatched weekdays — performed identically to the "right" weekdays, the cleanest possible refutation; (d) a pre-registered look-alike-analog study (862 days) passed 0 of 9 gates. The prediction line is kept as a seasonality display, labeled honestly. It does not predict.
What survived testing — the tool's actual value: movement magnitude and its clock. (1) The trailing-20-day average daily travel beats every fancier forecast of day size (median method's day-size miss ≈ 33 pts vs ~70 for the worst). (2) Sessions finish roughly half as far as they travel — the bridge from "expected travel" to what a finish-settled position (straddle/butterfly) actually pays. (3) The |move| U-shape by time-of-day and "Monday has the calmest afternoons" both held in split-half validation.

2 · Data & construction

  • Base data: SPX 1-minute bars (Schwab-fed, 2022-06 →), folded into 78 five-minute slots per session (09:30–15:55 ET); each slot's value = net SPX points open→close of the slot. Dataset self-feeds: the worker appends every session at EOD.
  • Prediction line: for a chosen weekday, take the last N same-weekdays (N ∈ 4…52, holidays auto-skipped, optional Fed/CPI/OPEX exclusion) and combine per slot by Sum / Average / Median / Volatility-weighted. Cumulative sum of slot values = the line.
  • Agreement band ("the cloud"): per-slot min/max across the N source weeks. A wide band straddling zero = the source weeks disagreed — the line is declared noise for that stretch (shaded faint red); a narrow one-sided band = the weeks rhymed (shaded green). This is an honesty overlay, not a signal.
  • Ghost lines: the individual source weeks drawn faintly, so one outlier week dragging the composite is visible at a glance.
  • Live overlay: today's actual cumulative line updates through the session (worker 5-min capture), with the day scored HIT/MISS on direction at the close — the running, unerasable record of §1's negative result.
line(slot) = combine( week₁(slot) … week_N(slot) ) # per 5-min slot band(slot) = [min, max] over the N weeks travel = mean over last 20 sessions of (high − low) # "expected movement" card finish ≈ 0.5 × travel # validated rule of thumb

Why Median is the default combine: on 990 scored days it produced the smallest day-size error (~33 pts). Sum is kept for illustration but draws the same line ~4× too tall by construction. Direction is a coin flip on all four methods, so the honest-scale one wins.

3 · The movement-regime table (the tradable part)

Because finish ≈ half of travel, the trailing-20 travel number maps to option-selling vs option-buying weather. Using 26 years of daily data (1,010 recent days scored):

4 · Validation protocol (why the negative result is trustworthy)

  • Controls before conclusions: the wrong-weekday control is the core device — if "Mondays predict Mondays" can't beat "Fridays predict Mondays," weekday identity carries no information, whatever the backtest curve looks like.
  • Baseline discipline: every method is scored against always-up and flat-zero baselines; beating history is meaningless unless the baseline loses.
  • Pre-registration: the analog study's 9 pass/fail gates were written down before the test ran — no post-hoc goalpost moves. It failed all 9; the result stands.
  • Split-half persistence: anything displayed as a keeper (U-shape, Monday afternoons, regime table rows) must hold in both halves of the sample independently.
  • Live scoreboard: direction HIT/MISS accumulates on-page daily, so the coin-flip verdict keeps re-proving itself in public.

5 · Limitations

  • Travel is measured high-to-low on 1-min closes — true tick range runs ~8% higher; wicks and overnight gaps are not captured.
  • The regime table's tails thin out at extremes (the "over 75" straddle cell is small-N in the live-strategy overlay).
  • Magnitude regimes are persistence-based (trailing 20) — they adapt slowly to vol shocks by construction; a Fed-day explosion lands in the average only afterward.
  • All findings are SPX-specific; the NDX toggle recomputes but has a shorter validated history.
