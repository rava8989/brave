GEX Dashboard — Complete Methodology

SPX dealer gamma exposure, live · ← back to the dashboard

1 · What this page measures

Option market-makers (dealers) hedge the options they hold by trading the underlying. The rate at which their hedge changes as price moves is gamma. Aggregated across the whole SPX option book, dealer gamma predicts the character of price action: long dealer gamma (PIN) → hedging sells rallies and buys dips → moves are dampened; short dealer gamma (BREAKOUT) → hedging chases price → moves are amplified. The dashboard estimates that book every minute of the session from the live Schwab SPX chain — no other data source.

2 · Core computation (OI basis)

per contract, strikes within the ±8% band (actual coverage displayed on-page): Γ = Black-Scholes gamma(S, K, T, IV) # 0DTE: T = real hours-to-close, 15-min floor GEX = Γ · OI · 100 · S² · 0.01 # dollars of hedging per 1% move dealer sign: calls → +GEX, puts → −GEX # dealers long calls / short puts netGEX(strike) = callGEX + putGEX; totalGEX = Σ over the band
  • Flip = zero-crossing of cumulative net GEX nearest spot, with a 2% significance gate: the cumulative must hold ≥2% of |totalGEX| on both sides, else the book is one-sided and no flip is reported. Computed over OI-bearing strikes only.
  • Max positive gamma = the pin/magnet strike; max negative = the wall. Walls list = top strikes by |netGEX|.
  • Regime = PIN if totalGEX > 0, BREAKOUT if < 0.
  • Vanna (VEX) and charm (CEX) use the same OI·S²·multiplier base: vanna = how the dealer book's delta shifts per +1 vol point; charm = delta drift from time decay (0DTE charm normalized per-day so it doesn't artificially decay into the close).

3 · The OI / Flow toggle — two estimates of one book

Why two bases exist: open interest updates only overnight (OCC). Intraday, OI-based GEX describes yesterday's book — nearly blind to 0DTE contracts born and dying the same day. The Flow basis re-signs today's traded volume by aggressor side to estimate the live book. Verified live 2026-06-26: OI read −$33B/BREAKOUT while Flow read +$76–121B/PIN, matching volume-aware third-party tools — the OI read was the stale one.
side = +1 if last ≥ ask (customer bought), −1 if last ≤ bid (sold), else mid-rule GEXvol = Γ · (−side · volume) · S² · 100 · 0.01 # dealer takes the OTHER side: customer buys any option → dealer short gamma

Honest caveat: this is a snapshot proxy, not tick-level Lee–Ready classification — each contract's full-day volume is signed by its current last-vs-quote. Coarse, but free and SPX-native. Rule of thumb: OI = structural, multi-day walls; Flow = the live 0DTE read. Neither is "the truth"; every GEX number is model-dependent.

5 · Data pipeline & integrity guards

  • Worker cron computes every minute of market hours from a self-fetched wide chain (150 strikes ≈ ±7.9%); the page displays actual strike coverage, never a claimed band.
  • Dead-chain guard: a chain with zero OI and zero volume everywhere writes nothing (prevents poisoning the baselines).
  • New contracts appearing mid-day enter as baselines, not as fake volume spikes.
  • Signed flow rolls up into a permanent daily dataset (signed_flow_daily) with weekly backups — the multi-week record used for edge testing.

6 · Research verdicts baked into this page

  • Flow vs OI for same-day 0DTE: use Flow; OI for persistent walls (§3).
  • Long-horizon seasonal tables are not a gate for 0DTE strategies — tested 2026-06-28, correlations ≈ 0 across all six strategies; horizons don't connect.
  • Day-type P&L DNA is descriptive, not predictive — realized move classes explain which strategy paid, but the class is only known at the close.

7 · Limitations

  • The dealer-sign convention (long calls / short puts) is an assumption, standard but unproven per-day; flip/walls inherit its error.
  • OI lags one day by construction; Flow's side-classification is a snapshot proxy (§3).
  • IV gaps fall back to 0.2; deep-ITM greeks are noisy.
  • GEX levels describe propensity (dampen vs amplify), not direction — the page never claims a directional forecast.
